Installation Process of Jekyll

By Alex
Posted 2022-05-28 Updated 2022-08-29 1 min read

Installation of Ruby and other requirements:

1

sudo apt-get install ruby-full build-essential zlib1g-dev

Configuration of Ruby Gems (do not want to install them as root user)

1
2
3
4

echo '\n# Install Ruby Gems to ~/gems' >> ~/.bashrc
echo 'export GEM_HOME="$HOME/gems"' >> ~/.bashrc
echo 'export PATH="$HOME/gems/bin:$PATH"' >> ~/.bashrc
source ~/.bashrc

Install bundler

1

gem install jekyll bundler

Running Jekyll-Based site

You will want to run the command:

1

bundle

just to make sure that gems dependencies are installed and checked. To execute and serve a Jekyll site, run:

1

bundle exec jekyll serve

Notes:

You may have to change the permissions of the directory you hold your repo in. If it looks like:

  • drwxrwxrwx 1 root 197609 0 Jun 18 22:23 {placeholder}.github.io/

You will need to use:

  • sudo chown -R $USER:$USER <filepath>

If you would like to generate the markup for a site, it is easy! The command will be:
JEKYLL_ENV=production bundle exec jekyll build
This will generate a file called _site which will be the folder you would want to serve locally.
